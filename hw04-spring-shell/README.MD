# Testing Students Application (Spring Shell Edition)

> Консольное приложение на Spring Boot для проведения тестирования студентов с поддержкой интерактивного интерфейса и локализации.

## О проекте

Это приложение позволяет проводить тестирование студентов через интерактивную консоль (Shell). Программа загружает вопросы из внешнего источника (CSV), взаимодействует с пользователем, проверяет ответы и выводит результат (Сдал/Не сдал) на основе настроек.

**Ключевые особенности:**
*   **Интерактивный CLI:** Реализован на **Spring Shell**.
*   **Локализация:** Поддержка нескольких языков (вывод вопросов и интерфейса в зависимости от настроек).
*   **Гибкая конфигурация:** Количество вопросов для сдачи и язык настраиваются через `application.yml`.
*   **AOP Логирование:** Использование **Spring AOP** для замеров времени выполнения методов и аудита действий студента.

## Технологический стек

*   **Core:** Java 21, Spring Boot
*   **Interface:** Spring Shell (интерактивные команды)
*   **Aspects:** Spring AOP (AspectJ)
*   **Utils:** Lombok, Apache Commons CSV (для чтения вопросов)
*   **Testing:** JUnit 5, Mockito

## Как запустить

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/olga-pegushina/student-testing-system.git
    cd student-testing-system
    ```

2.  **Соберите проект**

    либо с помощью инструментов IDE (Maven), либо используйте следующий вариант:

Для сборки проекта используется **Maven Wrapper** — это специальный скрипт (`./mvnw`), который позволяет собрать проект с правильной версией Maven, даже если он не установлен в вашей системе.

Выполните в корневой папке проекта:
```bash
./mvnw clean package
```
Для Windows используйте `mvnw.cmd clean package`.

> **‼️ Если команда `./mvnw` не найдена (ошибка "No such file or directory")**
>
> Это означает, что файлы Maven Wrapper не были добавлены в репозиторий. Чтобы их сгенерировать, у вас в системе должен быть установлен **Maven**.
> 1.  [Установите Apache Maven](https://maven.apache.org/install.html), если его нет.
> 2.  Выполните команду для генерации Wrapper'а:
      >     ```bash
      >     mvn wrapper:wrapper
      >     ```
> 3.  После этого файлы `mvnw` и `mvnw.cmd` появятся в проекте, и вы сможете вернуться к сборке командой `./mvnw clean package`.


3.  **Запустите JAR файл:**

### Запуск из среды разработки IntelliJ IDEA

Внутренняя консоль IntelliJ IDEA по умолчанию корректно работает с кодировкой UTF-8, что избавляет от необходимости дополнительной настройки.

## Важно: Настройка кодировки в IntelliJ IDEA

Если в консоли вместо кириллицы появляются `<?>` или "квадратики", выполните следующие настройки в IDEA, чтобы всё отображалось корректно.
### 1. Глобальные настройки кодировки

Это нужно, чтобы сама IDE правильно читала и сохраняла файлы.
1.  Откройте `File` -> `Settings...` (или `Ctrl+Alt+S`).
2.  Перейдите в `Editor` -> `File Encodings`.
3.  Установите везде **UTF-8**:
    *   **Global Encoding**: `UTF-8`
    *   **Project Encoding**: `UTF-8`
    *   **Default encoding for properties files**: `UTF-8`
    *   Убедитесь, что стоит галочка `Create UTF-8 files with NO BOM`.

### 2. Настройка для запуска приложения
Чтобы Java Virtual Machine (JVM) использовала UTF-8 при запуске вашего приложения.
1.  Откройте конфигурацию запуска: `Run` -> `Edit Configurations...`.
2.  Выберите вашу конфигурацию для Spring Boot приложения.
3.  Найдите поле `Modify options` и в выпадающем списке выберите `Add VM options`.
4.  В появившемся поле **VM options** вставьте строку:
    ```
    -Dfile.encoding=UTF-8
    ```
5.  Нажмите `Apply`, затем `OK`.

После этих настроек перезапустите приложение. Проблема с кодировкой должна исчезнуть!

## Использование (Shell команды)

После запуска приложения вы попадете в интерактивную оболочку. Доступные команды:

### 1. Авторизация
Сначала необходимо представиться системе.
```shell
shell:> login
# или коротко
shell:> l
```
_Система запросит имя и фамилию._

### 2. Запуск теста
Тестирование доступно только после входа (команда `login`).
```shell
shell:> start
# или коротко
shell:> s
```

### 3. Выход
Чтобы завершить работу с приложением.
```shell
shell:> quit
# или коротко
shell:> q
```

## Настройки (application.yml)
Вы можете изменить параметры тестирования в файле конфигурации `src/main/resources/application.yml`:

```yaml
test:
  # Количество правильных ответов для зачета
  rightAnswersCountToPass: 9
  # Локаль по умолчанию (ru-RU или en-US)
  locale: ru-RU
  # Пути к файлам с вопросами (CSV) для каждой локали
  fileNameByLocaleTag:
    ru-RU: questions_ru.csv
    en-US: questions.csv

# Настройки локализации Spring
spring:
  messages:
    baseName: messages
    encoding: UTF-8
    fallbackToSystemLocale: false
```

## Архитектура
*   **Service Layer:** Основная бизнес-логика (`TestServiceImpl`, `ResultServiceImpl`).
*   **Shell:** Обработка команд пользователя (`ShellTestCommands`).
*   **Aspects:** Сквозная функциональность:
    *   `LoggingAspect` — логирует время выполнения методов, отмеченных `@Loggable`.
    *   `StudentActionAspect` — логирует вход студентов и ошибки в сервисах.
*   **Security:** Простой `InMemoryLoginContext` для контроля доступа к команде старта теста.
